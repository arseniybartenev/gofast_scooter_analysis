## Описание проекта

В проекте проведён всесторонний анализ данных сервиса аренды самокатов **GoFast**. Исследованы поведенческие паттерны пользователей, структура поездок и влияние подписки `ultra` на ключевые метрики. На основе статистических тестов проверены гипотезы, позволяющие бизнесу принимать обоснованные решения по развитию и продвижению платной подписки.

**Цель исследования:**  
Определить достоверность гипотез о поведении пользователей с подпиской и без неё для дальнейшего увеличения числа подписчиков и роста выручки.

## Данные

Исходные данные представлены тремя CSV-файлами:

- `users_go.csv` – информация о пользователях: уникальный идентификатор, имя, возраст, город, тип подписки (free / ultra).
- `rides_go.csv` – данные о поездках: идентификатор пользователя, расстояние (в метрах), продолжительность (в минутах), дата.
- `subscriptions_go.csv` – условия подписок: тип подписки, стоимость минуты, стоимость старта, абонентская плата.


## Этапы работы

1. **Загрузка и первичный анализ**  
   - Импорт библиотек, загрузка трёх датасетов, общая информация, описательные статистики.

2. **Предобработка данных**  
   - Проверка пропусков (отсутствуют).  
   - Удаление дубликатов из таблицы пользователей (31 строка).  
   - Приведение столбца `date` к типу datetime и добавление поля `month`.

3. **Исследовательский анализ данных (EDA)**  
   - Визуализация распределения пользователей по городам, возрастам.  
   - Анализ расстояния и продолжительности поездок.  
   - Сравнение характеристик поездок для пользователей с подпиской `ultra` и без подписки.

4. **Объединение данных и расчёт выручки**  
   - Слияние всех трёх таблиц в единый датафрейм.  
   - Группировка по пользователю и месяцу: суммарное расстояние, суммарная длительность, количество поездок.  
   - Расчёт помесячной выручки по каждому пользователю с учётом условий подписки.

5. **Проверка гипотез**  
   - Сравнение продолжительности поездок между группами (ttest_ind, альтернатива greater).  
   - Проверка среднего расстояния пользователей с подпиской против оптимального значения 3130 м (ttest_1samp).  
   - Сравнение средней месячной выручки от подписчиков и пользователей без подписки (ttest_ind).  
   - Теоретический вопрос о выборе критерия для зависимых выборок (ttest_rel).

6. **Формулировка выводов и рекомендаций**

## Результаты

| Проверяемая гипотеза | Результат | Значимость |
|----------------------|-----------|------------|
| Время поездки у пользователей с подпиской больше, чем у пользователей без подписки | ✅ отвергнута H₀ | p‑value ≈ 1.28e-34 |
| Среднее расстояние за одну поездку у подписчиков превышает 3130 м | ❌ не удалось отвергнуть H₀ | p‑value ≈ 0.92 |
| Помесячная выручка от подписчиков выше, чем от пользователей без подписки | ✅ отвергнута H₀ | p‑value ≈ 1.89e-31 |

- Пользователи с подпиской `ultra` в среднем тратят больше времени на поездку и приносят значительно бóльшую месячную выручку.
- Среднее расстояние за поездку у подписчиков не превышает оптимальный с точки зрения износа самоката показатель (3130 м), что подтверждает эффективность текущей модели использования.

## Выводы

Проведённый анализ позволяет рекомендовать **активное продвижение подписки `ultra`**, так как:

- Подписчики генерируют существенно более высокую выручку по сравнению с пользователями бесплатного тарифа.
- Их поведение (время и дистанция поездок) не приводит к повышенному износу самокатов, оставаясь в пределах оптимальных значений.
- Внедрение подписки способствует росту лояльности и увеличению частоты использования сервиса.

## Запуск проекта

1. Клонируйте репозиторий:
   ```bash
   git clone https://github.com/arseniybartenev/gofast_scooter_analysis.git
   ```
2. Перейдите в папку проекта:
   ```bash
   cd gofast_scooter_analysis
   ```
3. Установите зависимости:
   ```bash
   pip install -r requirements.txt
   ```
4. Запустите Jupyter Notebook:
   ```bash
   jupyter notebook notebooks/analysis_of_a_scooter_rental_service.ipynb
   ```
